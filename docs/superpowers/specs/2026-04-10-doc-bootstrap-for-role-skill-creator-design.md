# role-skill-creator：文档缺失时的代码整理与缓存文档

日期：2026-04-10  
状态：已评审通过（用户确认「好」）

## 1. 目标

在 `role-skill-creator` 中增加能力：当目标代码仓库**缺少可读文档、主要仅剩代码**时，能够：

1. **主动**整理代码结构层面的信息（非替代完整人工文档）。
2. 在**目标仓库内**生成可版本管理的缓存文档目录。
3. 将**指向该缓存的引用**写入生成的角色 `SKILL.md`，便于后续按需查询（渐进披露）。
4. 支持多仓库：每个仓库独立一套生成物。

## 2. 范围

### 2.1 纳入

- 文档稀疏度检测与自动进入「文档引导」流程（可关闭）。
- 默认落盘于目标仓库：`docs/agent-skill-cache/`，可通过 `doc_output_root` 或按仓映射覆盖。
- 生成 `INDEX.md`（人读）与 `manifest.json`（机读/Agent 筛选）。
- 生成的角色 skill 中增加「项目文档缓存」指针与读取策略。
- 幂等更新、敏感信息处理原则、评测断言扩展。

### 2.2 不纳入（本阶段）

- 替代团队正式架构文档或对外 API 文档体系。
- 全语言/全框架的完备静态分析器（仅允许轻量启发式 + 模型摘要）。
- 默认修改目标仓库的 `.gitignore`（可选说明，不自动写入除非后续明确需求）。

## 3. 方案选择

采用 **混合方案（轻量确定性扫描 + 模型生成摘要 + 固定 INDEX/manifest 契约）**，在稳定性与表达力之间平衡，便于迭代与评测。

## 4. 行为规格

### 4.1 触发条件（文档稀疏）

对每个 `target_repo_paths` 中的仓库，使用启发式判断（可配置阈值，首版可用规则组合）：

- 根 `README` 缺失或过短；
- 无 `docs/` 或 `docs/` 几乎为空；
- 缺少架构/模块说明类文件；

**命中则默认自动进入**「代码文档化引导」流程。

### 4.2 关闭自动写盘

以下任一成立则**不写盘**，仅分析或仅输出大纲（与现有「只读分析」约束兼容）：

- 输入 `doc_bootstrap: false`；或
- `constraints` 包含 `skip-doc-bootstrap`。

### 4.3 落盘路径

- **默认**：`<repo-root>/docs/agent-skill-cache/`
- **覆盖**：`doc_output_root`（相对该仓库根）；多仓映射：`doc_output_roots: { "<abs-path>": "relative/path" }`，未列出的仓库使用默认。

### 4.4 目录与文件契约

每个仓库下建议最小结构：

```
docs/agent-skill-cache/
├── INDEX.md           # 导航与主题索引
├── manifest.json      # 路径、标签、摘要、源码指针、生成元数据
└── （可选）modules/   # 按模块拆分的 md，按需增量
```

`manifest.json` 建议字段（首版可精简，后续演进需 `schema_version`）：

- `schema_version`
- `generated_at`
- `repo_root` 或仓库标识（与 `repo_aliases` 对齐）
- `entries[]`: `path`, `title`, `summary`, `tags`, `source_paths[]`

### 4.5 并入生成的角色 Skill

在产出的角色 `SKILL.md` 中增加一节（名称可统一为 **「项目文档缓存」**）：

- 写明相对**该目标仓库根**的路径，例如：`docs/agent-skill-cache/INDEX.md`
- 说明 **何时读取**：回答架构/模块/边界问题前，优先读 `INDEX.md` 或按 `manifest.json` 定位
- **禁止**把整份摘要塞进 `SKILL.md` 正文；仅指针 + 策略（渐进披露）

多仓库时，按 `repo_aliases` 分节列出各仓缓存路径，避免混淆。

### 4.6 幂等与敏感信息

- **幂等**：重复运行应更新 `manifest` 与 `generated_at`，覆盖或合并策略明确为「同路径文件以最新生成为准」，避免无限追加重复段落。
- **敏感信息**：生成前提示跳过或脱敏常见密钥模式；不在文档中复述 `.env` 等内容；命中标为需人工确认。

### 4.7 错误处理

- 目标路径不可写：报错并说明权限；不部分写入导致不一致。
- 仅部分仓库可写：按仓库粒度报告成功/失败，不静默吞掉。

## 5. 评测

在 `role-skill-creator` 的 `evals` 中增加用例：

1. **文档稀疏 + 默认行为**：断言生成 `INDEX.md` 与 `manifest.json`（路径符合约定）。
2. **关闭 doc bootstrap**：断言不写盘或明确跳过写盘步骤。
3. **生成的角色 skill**：断言包含对 `docs/agent-skill-cache/` 的可解析引用。

## 6. 与现有能力的关系

- 多仓库冲突策略（先提示、不自动定夺）**不变**；文档缓存与冲突清单可并列输出。
- `references/input-schema.md` 需同步扩展字段说明（后续实现任务）。

## 7. 验收标准

- 设计评审通过且本文件无 TBD。
- 实现后：稀疏文档场景下能生成约定结构；关闭开关时不污染仓库；角色 skill 能指向缓存并支持后续查询。
